外部推送 API

允许外部应用推送新闻至工作台数据库

您的 API 密钥

点击获取密钥

请妥善保管您的 API 密钥，不要分享给他人

POST /api/v1/news/push/

推送单条新闻到工作台

# 请求示例
curl -X POST "https://your-domain.com/api/v1/news/push/" \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "新闻标题",
    "link": "https://example.com/article/123",
    "content": "新闻正文内容...",
    "summary": "新闻摘要（可选）",
    "author": "作者名（可选）",
    "category": "分类（可选）",
    "image_url": "https://example.com/image.jpg（可选）",
    "published_at": "2024-01-15T10:30:00Z（可选，ISO 8601 格式）",
    "source_name": "来源名称（可选）"
  }'

请求参数

参数 类型 必填 说明
title string 是 新闻标题
link string 是 原文链接
content string 否 新闻正文内容
summary string 否 新闻摘要
author string 否 作者名
category string 否 分类
image_url string 否 图片链接
published_at string 否 发布时间（ISO 8601 格式）
source_name string 否 来源名称
POST /api/v1/news/push/batch/

批量推送多条新闻到工作台

# 请求示例
curl -X POST "https://your-domain.com/api/v1/news/push/batch/" \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "source_name": "批量推送来源（可选）",
    "articles": [
      {
        "title": "新闻标题1",
        "link": "https://example.com/article/1",
        "content": "内容1..."
      },
      {
        "title": "新闻标题2",
        "link": "https://example.com/article/2",
        "content": "内容2..."
      }
    ]
  }'
GET /api/v1/status/

检查 API 服务状态

# 请求示例
curl "https://your-domain.com/api/v1/status/"

响应格式

# 成功响应
{
  "success": true,
  "message": "News article pushed successfully",
  "data": {
    "id": 123,
    "title": "新闻标题",
    "created": true,
    "published_at": "2024-01-15T10:30:00+00:00"
  }
}

# 错误响应
{
  "error": "Missing API key",
  "code": 401
}

错误码

状态码 说明
200 请求成功
400 请求参数错误
401 缺少 API 密钥
403 API 密钥无效
404 用户不存在
